# Download Page Redesign — Design Spec

**Date:** 2026-06-24
**Goal:** Переработать `/download` по структуре референса (TLauncher), адаптировав контент под SBGames-лаунчер. В нашем тёмном дизайне. SEO-оптимизация под топ-1.

## Решения (из брейншторма)

- **Модалка скачивания** открывается **по клику** на любую кнопку «Скачать». Внутри — автоопределение устройства по `navigator.userAgent`. Авто-попапа нет.
- **Hero** остаётся простым (как сейчас), не переименовываем в «SBGames Launcher».
- **Блок «все версии → все серверы»** убран (серверы уже на главной).
- **Блок «моды/модлоадеры в один клик»** — оставляем **1:1 как на референсе** (Forge/Fabric/Optifine).
- Подход реализации — **B**: лендинг + отдельный компонент модалки + хук определения устройства.

## Структура страницы (порядок секций)

1. **Hero** — простой: заголовок «Скачать лаунчер» + подзаголовок + бейдж версии + кнопка «Скачать» (открывает модалку)
2. **Секция фич: «Установи Forge, Fabric, Optifine в один клик»** — 1:1 по референсу
3. **Системные требования**
4. **FAQ**
5. **Footer**

## Архитектура (новые/изменяемые файлы)

```
website/src/
├── pages/DownloadPage.jsx          ← переписать: лендинг
├── components/
│   ├── DownloadModal.jsx           ← новый: модалка с автоопределением ОС
│   └── DownloadTrigger.jsx         ← новый: обёртка-кнопка, открывает модалку
└── lib/
    └── useDeviceDetect.js          ← новый: хук определения ОС по userAgent
```

- `DownloadPage` владеет state `isModalOpen` и передаёт `openModal`/`closeModal` вниз.
- `DownloadTrigger` — универсальная кнопка «Скачать», дёргает `openModal`.
- `useDeviceDetect()` возвращает `{ platform: "windows" | "macos" | "linux" | "android" | "ios", isMobile }`.

## Детали секций

### Hero
- Заголовок: «Скачать лаунчер» (как сейчас)
- Подзаголовок: «Выбери свою платформу — установи и играй» (как сейчас)
- Бейдж версии (если есть манифест)
- Кнопка «Скачать» → `openModal()` (вместо прямого скачивания)
- Дизайн: тёмный фон, `rgba(255,255,255,0.0x)` карточки, framer-motion fade-in

### Секция фич: моды/модлоадеры в один клик
Контент из референса, адаптированный:
- Заголовок: **«УСТАНОВИ FORGE, FABRIC, OPTIFINE В ОДИН КЛИК»**
- Подзаголовок: про автоустановку модификаций под каждый сервер без ручной настройки
- Левая/правая колонки: список преимуществ + скриншот/иллюстрация
- Пункты преимуществ (адаптация):
  - Forge — для тяжёлых модов и серверов
  - Fabric — лёгкий, быстрый, современные моды
  - Optifine — оптимизация и шейдеры
  - Авто-подбор версии под сервер (нажал «Играть» — лаунчер сам скачал нужное)
  - Обновления в фоне
- Внутри секции кнопка «Скачать» → `openModal()`

### Системные требования
- Карточка с минимальными/рекомендуемыми требованиями
- Колонки: Минимально / Рекомендуется
- Поля: ОС, RAM, CPU, диск, Java
- Значения (ориентир): Win 10+, 4/8 ГБ RAM, Java 17+

### FAQ
- Аккордеон (раскрывающиеся вопросы), `framer-motion` для анимации
- Вопросы (SEO-ориентированные):
  - Безопасно ли скачивать лаунчер?
  - Это бесплатно?
  - Какие нужны системные требования?
  - Как обновить лаунчер?
  - Что делать, если не запускается?
  - Нужно ли отдельно ставить Java/Forge?
  - Антивирус ругается — что делать?
- FAQ обёрнут в **JSON-LD `FAQPage`** для rich snippets в поиске (SEO).

### Footer
- Как на HomePage: `© 2026 SBGames. All rights reserved.`

### Модалка скачивания (`DownloadModal.jsx`)
- Overlay: затемнённый фон + блюр, клик вне → закрытие, Esc → закрытие
- Контент (в нашем тёмном дизайне, не белый как у референса):
  - Заголовок: «Скачать SBGames»
  - Строка автоопределения: иконка устройства + «Ваше устройство: **Windows**» (подсвечено)
  - Главная кнопка под определённую ОС (цвет платформы) → прямой `download` ссылки
  - Версия + размер файла
  - Разделитель «или выбери платформу вручную»
  - Переключатели Win/macOS/Linux (как сейчас)
  - Кнопка закрытия (X)
- Данные: из того же `latest.json` манифеста
- Device detection при открытии модалки (не при загрузке страницы)

## SEO (максимальная оптимизация)

### Title / meta (обновить под /download)
- `<title>`: «Скачать лаунчер SB Games — Minecraft серверы | Windows, macOS, Linux»
- `<meta description>`: с ключевыми словами (лаунчер, скачать, minecraft, серверы, forge, бесплатно, windows)
- `<meta keywords>`: расширенный набор

### JSON-LD (добавить в index.html или инжектить на /download)
- `SoftwareApplication` (уже есть — обновить: version, fileSize, operatingSystem через несколько ОС)
- `FAQPage` — для rich snippets (вопросы из секции FAQ)
- `BreadcrumbList` — Главная / Скачать

### Семантическая разметка
- `<h1>` один на странице (Hero)
- `<h2>` для секций, `<h3>` для пунктов
- `<section aria-label>` для каждой секции
- `alt` у изображений с ключевыми словами
- Документ в `<main>`, секции в `<section>`
- Болен содержательный текст (не только заголовки) — поисковики любят текст про лаунчер/моды/установку

### Производительность (фактор ранжирования)
- `loading="lazy"` на изображения секций (не Hero)
- `fetchpriority="high"` на Hero-изображение если есть
- preconnect к `games.sb-capital.group` (откуда манифест)

## Адаптив
- Mobile-first: на узких экранах колонки фич-секции становятся в столбец, таблица требований — карточки.
- Модалка: на мобиле на весь экран (full-screen sheet), на десктопе — центральная 420px карточка.

## Данные
- Манифест `https://games.sb-capital.group/downloads/latest.json` — без изменений (версия, дата, платформы, размеры).
- Device detection — клиентский, без бэкенда.

## Безопасность / edge cases
- Если манифест не загрузился → показываем кнопки с дефолтной версией, без размера.
- Если платформа не определена → дефолт Windows, пользователь может переключить вручную.
- Блокировка скролла body при открытой модалке.
- Фокус-трап внутри модалки (a11y).
