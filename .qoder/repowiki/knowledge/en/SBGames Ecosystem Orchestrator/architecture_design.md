- The root module orchestrates the ecosystem by defining shared build contracts: the `build:java` script generates an obfuscated Java bootstrap (`sbg-bootstrap.jar`) consumed by both the desktop client and backend verification services.
- It manages a dual-frontend strategy where the primary Vite build targets the Tauri desktop app (`dist/`) and a secondary build targets the system tray (`dist-tray/`), while a separate `website/` module handles the public-facing portal.
- Cross-module communication is standardized via a centralized Node.js/Express backend (`server_index.js`) that serves as the single source of truth for authentication (Telegram-based), real-time WebSocket events, and inventory management for both the desktop and web clients.
- Deployment is unified through root-level CI/CD configurations (`.cirrus.yml`, `.gitlab-ci.yml`) and shell scripts (`scripts/build-all.sh`) that coordinate Rust/Tauri builds for the desktop app and Node.js provisioning for the backend.
