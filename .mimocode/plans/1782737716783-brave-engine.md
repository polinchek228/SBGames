# Оптимизация скорости скачивания в лаунчере

## Цель
Сделать скачивание ресурсов сборок максимально быстрым — близко к лимиту пропускной способности интернета.

## Текущая проблема
При создании сборки/запуске игры скачивание длится минутами. Основные причины:
- 2000+ ассетов скачиваются **по одному**
- 50+ модов скачиваются **по одному**
- Серверные модпаки скачиваются через **serial curl** (по одному)
- HTTP/2 не включён — каждое соединение = отдельный TLS-хендшейк
- После скачивания каждого ассета — повторное чтение файла для SHA1-проверки (двойной I/O)
- Ассеты и библиотеки скачиваются **последовательно**, хотя они независимы

## План (по приоритету impact)

### Фаза 1: Фундамент

**1.1 Включить HTTP/2** — `src-tauri/Cargo.toml`
- Добавить feature `"http2"` к reqwest
- Эффект: все параллельные скачивания шарят одно TCP+TLS соединение через мультиплексирование
- Сложность: простая

**1.2 Увеличить пул соединений** — `src-tauri/src/lib.rs:2751`
- `pool_max_idle_per_host`: 16 → 32
- `pool_idle_timeout`: 90s → 300s
- Сложность: простая

**1.3 Убрать дублирование HTTP-клиентов** — `vanilla.rs`, `mods.rs`
- `http_get_json` и `resolve_modrinth_version` создают отдельный `reqwest::Client` на каждый вызов
- Заменить на использование `SHARED_CLIENT`
- Сделать `SHARED_CLIENT` публичным (`pub(crate)`)
- Эффект: ~5s экономии на 50 модах (100ms TLS × 50)
- Сложность: простая

### Фаза 2: Параллелизация (главный выигрыш)

**2.1 Параллельная загрузка ассетов** — `src-tauri/src/loaders/vanilla.rs:146`
- Заменить serial `for` на `stream::iter().map().buffer_unordered(16)`
- 2000+ мелких файлов: serial ~100s → parallel(16) ~13s
- Сложность: средняя

**2.2 Параллельная загрузка модов** — `src-tauri/src/mods.rs:28-60`
- Заменить serial `for` на `stream::iter().map().buffer_unordered(8)`
- 50 модов: serial ~10s → parallel(8) ~1.3s
- Сложность: средняя

**2.3 Заменить curl на parallel reqwest** — `src-tauri/src/lib.rs:2239-2313`
- Серверный синк модпака спавнит curl на каждый мод
- Заменить на `download_file` через `buffer_unordered(8)`
- Убирает overhead запуска процесса (~50ms на curl) + даёт параллелизм
- Сложность: средняя

**2.4 Параллельная загрузка mrpack** — `src-tauri/src/mrpack.rs:105`
- Аналогично — `buffer_unordered(8)`
- Сложность: средняя

### Фаза 3: Устранение лишнего I/O

**3.1 Хеширование на лету** — `src-tauri/src/loaders/vanilla.rs`
- Вместо "скачать → перечитать для SHA1" — считать SHA1 **во время** скачивания
- Новая функция `download_file_hashed`, обёртка над `download_file`
- Убирает повторное чтение ~2000 файлов
- Сложность: средняя

**3.2 Параллельная загрузка библиотек + ассетов** — `src-tauri/src/loaders/vanilla.rs:102-107`
- `ensure_profile_libraries` и `ensure_assets` независимы → `tokio::join!`
- Экономия ~5-10s
- Сложность: простая

### Фаза 4: Прогресс-бар

**4.1 Агрегированный счётчик** — `vanilla.rs`, `mods.rs`
- `AtomicU64` счётчик завершённых файлов
- Emit каждые 300ms вместо каждого файла
- Убирает лаг UI при 16+ параллельных скачиваниях
- Сложность: средняя
